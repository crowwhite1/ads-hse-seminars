# A. Однажды на паре

## Условие

Дан массив `a` длины `n`.

Не более `k` раз можно выполнить операцию:

- выбрать индексы `i` и `j` (`1 <= i, j <= n`);
- индекс `j` **не должен использоваться как донор повторно**;
- присвоить `a[i] = |a[j]|`.

Нужно найти **максимально возможную сумму** элементов массива после не более чем `k` операций.


## Ключевая идея

Главное, что дает операция:  
за один ход мы можем **заменить один элемент массива** на **модуль какого-то другого элемента**.

Значит, хочется понять:

1. **какое значение выгоднее всего копировать;**
2. **какие элементы выгоднее всего заменять.**


## Наблюдение 1. Лучшее значение для копирования — это максимальный модуль

Обозначим

```text
M = max(|a1|, |a2|, ..., |an|)
````

Тогда никакая операция не может создать число больше `M`, потому что мы всегда присваиваем элементу значение вида `|a[j]|`, а модуль любого исходного/полученного числа не превосходит `M`.

Значит, **самое выгодное значение, которое вообще можно ставить в ячейки массива, — это `M`**.


## Наблюдение 2. Каждой операцией выгодно превращать какой-то элемент в `M`

Если мы меняем `a[i]` на `M`, то прирост суммы равен

```text
M - a[i]
```

Так как `a[i] <= M` для любого `i`, этот прирост всегда неотрицателен.

Следовательно:

* если у нас еще остались операции,
* и мы можем сделать элемент равным `M`,

то это никогда не ухудшает ответ.


## Наблюдение 3. Ограничение “индекс `j` нельзя использовать дважды” не мешает сделать до `k` элементов равными `M`

На первый взгляд может показаться, что если индекс-донор `j` нельзя использовать повторно, то значение `M` получится скопировать только один раз.

Но это не так.

### Почему?

Пусть есть хотя бы один индекс `p`, для которого

```text
|a[p]| = M
```

Тогда:

* на первом шаге можно сделать какой-то элемент равным `M`;
* этот новый элемент теперь сам содержит `M`;
* значит, на следующем шаге его уже можно использовать как нового донора;
* и так далее.

То есть значение `M` можно **“размножать по цепочке”**.

### Пример

```text
a = [-10, 1, 1], k = 2
M = 10
```

1. используем индекс `1` как донора и делаем `a[2] = |a[1]| = 10`
   получаем `[-10, 10, 1]`

2. теперь индекс `2` уже сам содержит `10`, используем его как донора
   и делаем `a[1] = |a[2]| = 10`
   получаем `[10, 10, 1]`

Итого за 2 операции мы сделали **два** элемента равными `M`, хотя исходно такой элемент был только один.


## Что тогда оптимально?

Если каждая операция позволяет превратить один элемент в `M`, то нужно выбирать те элементы, для которых прирост суммы максимален:

```text
прирост = M - a[i]
```

Чем меньше `a[i]`, тем больше прирост.

Значит, выгоднее всего менять **самые маленькие элементы массива**.


## Итоговая стратегия

1. Найти

   ```text
   M = max(|a[i]|)
   ```
2. Отсортировать массив по возрастанию.
3. Заменить первые `k` элементов на `M`.
4. Посчитать итоговую сумму.


## Почему это корректно

### Шаг 1. Больше `M` получить нельзя

Любая операция присваивает значение вида `|a[j]|`.
Следовательно, ни один элемент после операций не может стать больше `M`.


### Шаг 2. Если мы хотим улучшить сумму, то лучший выбор — поставить `M`

Если в некоторый элемент можно записать любое допустимое значение, то самое выгодное из них — максимальное, то есть `M`.


### Шаг 3. За каждую операцию оптимально менять один из минимальных элементов

Прирост при замене `a[i]` на `M` равен `M - a[i]`.
Этот прирост тем больше, чем меньше `a[i]`.

Следовательно, нужно выбирать `k` минимальных элементов массива.


### Шаг 4. Такое решение всегда достижимо

Значение `M` можно передавать по цепочке через уже измененные элементы, поэтому ограничение на уникальность донора `j` не мешает выполнить до `k` полезных замен.


## Формула ответа

Если после сортировки массива

```text
a[0] <= a[1] <= ... <= a[n-1]
```

то ответ равен

```text
M + M + ... + M   (k раз)
+ a[k] + a[k+1] + ... + a[n-1]
```

То есть мы просто заменяем `k` минимальных элементов на `M`.

Эквивалентно:

```text
answer = sum(a) + Σ(M - a[i]) по k минимальным элементам
```


## Пример 1

### Вход

```text
n = 5, k = 2
a = [-4, 2, 7, -1, 3]
```

### Шаг 1. Находим `M`

```text
M = max(4, 2, 7, 1, 3) = 7
```

### Шаг 2. Сортируем

```text
[-4, -1, 2, 3, 7]
```

### Шаг 3. Меняем первые `k = 2` элемента на `7`

```text
[7, 7, 2, 3, 7]
```

### Ответ

```text
7 + 7 + 2 + 3 + 7 = 26
```


## Пример 2

### Вход

```text
n = 3, k = 1
a = [-10, 1, 1]
```

```text
M = 10
```

Самый выгодный элемент для замены — `-10`.

Можно выбрать `i = j = 1` и получить:

```text
[10, 1, 1]
```

Сумма:

```text
12
```


## Пример 3

### Вход

```text
n = 4, k = 0
a = [1, -5, 2, 4]
```

Операции запрещены, ответ — просто сумма массива:

```text
1 + (-5) + 2 + 4 = 2
```


## Сложность

### По времени

* поиск `M`: `O(n)`
* сортировка: `O(n log n)`
* подсчет ответа: `O(n)`

Итого:

```text
O(n log n)
```

### По памяти

Если не считать массив:

```text
O(1)
```

Если считать хранение массива:

```text
O(n)
```


## Почему жадность здесь работает

Потому что:

* итоговая “ценность” каждого изменения независима от других и равна `M - a[i]`;
* все эти приросты неотрицательны;
* за одну операцию можно улучшить ровно один элемент;
* значит, надо брать `k` наибольших приростов, то есть `k` самых маленьких элементов.

Это классическая жадная идея:
**в каждый из доступных `k` слотов изменений кладем максимально выгодное улучшение**.


## Частые ошибки

### 1. Думать, что один и тот же максимальный элемент можно использовать как донор только один раз, значит `M` копируется только один раз

Нет.
После первой операции новый элемент тоже станет равен `M` и может сам выступить донором дальше.


### 2. Пытаться отдельно обрабатывать отрицательные и положительные числа

На самом деле это не нужно.
Достаточно понять, что каждый выбранный элемент выгодно заменить на `M`.


### 3. Пытаться выбрать элементы с максимальным модулем для замены

Менять нужно не элементы с большим модулем, а элементы с **минимальным значением**, потому что прирост равен именно `M - a[i]`.

## Краткий вывод

Вся задача сводится к одной мысли:

> каждая операция позволяет сделать один элемент равным максимально возможному значению `M = max(|a[i]|)`.

Поэтому оптимально просто:

* найти `M`,
* отсортировать массив,
* заменить `k` минимальных элементов на `M`.

Именно это дает максимальную возможную сумму.
