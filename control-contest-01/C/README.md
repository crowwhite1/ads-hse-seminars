# C. Стратегии успеха на БИ

## Идея задачи

Есть:

- `n` прямых вида `y = kx`,
- `m` парабол вида `y = ax² + bx + c`, где `a > 0`.

Для каждой параболы нужно понять, существует ли такое значение `k` из списка стратегий, что прямая:

```text
y = kx
````

**не пересекает** параболу и **не касается** её.

Если такая стратегия есть, нужно вывести `YES` и любой подходящий `k`.
Иначе — `NO`.

Решение из файлов делает именно это: сортирует все доступные `k`, а для каждой параболы ищет подходящий коэффициент бинарным поиском.  


## Геометрический смысл

Мы сравниваем графики:

```text
y = kx
y = ax² + bx + c
```

Точки пересечения удовлетворяют уравнению:

```text
ax² + bx + c = kx
```

Перенесём всё в одну сторону:

```text
ax² + (b - k)x + c = 0
```

Это квадратное уравнение относительно `x`.

Чтобы прямая **не пересекала** параболу и **не касалась** её, у этого уравнения **не должно быть действительных корней**.

Значит, нужен строгий отрицательный дискриминант:

```text
D < 0
```


## Вывод условия на `k`

Для уравнения

```text
ax² + (b - k)x + c = 0
```

дискриминант равен:

```text
D = (b - k)² - 4ac
```

Нам нужно:

```text
(b - k)² - 4ac < 0
```

то есть:

```text
(b - k)² < 4ac
```

Берём корень:

```text
|b - k| < 2√(ac)
```

Это равносильно:

```text
b - 2√(ac) < k < b + 2√(ac)
```

Именно этот интервал и вычисляется в решении: левая граница `kx_m = b - 2 * sqrt(a * c)`, правая — `kx_p = b + 2 * sqrt(a * c)`.  


## Что это значит на практике

Для каждой параболы задача сводится к очень простой проверке:

> Есть ли среди имеющихся значений `k` хотя бы одно число, строго лежащее внутри интервала

```text
(b - 2√(ac), b + 2√(ac))
```

Если есть — ответ `YES`.
Если нет — `NO`.


## Почему достаточно искать только первое число больше левой границы

После сортировки массива `k` можно бинарным поиском найти:

> первое значение `k`, которое **строго больше** левой границы.

В Python это сделано через `bisect_right`, а в Java — через собственный `upperBound`.  

Дальше достаточно проверить только это число:

* если оно ещё и **меньше правой границы**, то оно подходит;
* если уже не меньше правой границы, то никакое следующее число тоже не подойдёт, потому что массив отсортирован.

Это и есть ключевая оптимизация решения.  


## Откуда берётся бинарный поиск

Пусть после сортировки:

```text
k[0] <= k[1] <= ... <= k[n-1]
```

Для каждой параболы мы ищем первое число:

```text
k[pos] > leftBorder
```

Если такого числа нет, сразу `NO`.

Если есть, остаётся проверить:

```text
k[pos] < rightBorder
```

Именно поэтому решение работает за:

```text
O(n log n + m log n)
```

на один тест:

* сортировка всех `k` — `O(n log n)`,
* для каждой из `m` парабол один бинарный поиск — `O(log n)`.

Это соответствует коду в Python, Java и C++.   


## Важное наблюдение про `ac`

Чтобы интервал

```text
(b - 2√(ac), b + 2√(ac))
```

вообще существовал как ненулевой действительный интервал, нужно:

```text
ac > 0
```

Так как по условию `a > 0`, всё упирается в знак `c`.

### Если `c < 0`

Тогда `ac < 0`, а значит:

```text
D = (b - k)² - 4ac = (b - k)² + что-то_положительное > 0
```

То есть корни всегда есть, прямая обязательно пересечёт параболу.
Подходящей стратегии быть не может.

В Python-решении это отдельно обработано проверкой `if a * c < 0: NO`. 

### Если `c = 0`

Тогда условие становится:

```text
(b - k)² < 0
```

Это невозможно.
Значит, при `c = 0` ответ тоже всегда `NO`.

В Java-коде отдельной проверки нет, но общая формула всё равно корректно это обработает: интервал выродится в точку, и строгое попадание внутрь окажется невозможным. 


## Пошаговый алгоритм

Для каждого теста:

### 1. Считать все `k`

Сохраняем коэффициенты прямых в массив и сортируем его.
Это делается во всех трёх реализациях.   

### 2. Для каждой параболы `(a, b, c)` вычислить допустимый интервал

Ищем такие `k`, что:

```text
b - 2√(ac) < k < b + 2√(ac)
```

### 3. Найти первое `k > leftBorder`

Используем бинарный поиск:

* `bisect_right` в Python,
* `upperBound` в Java,
* `upper_bound` в C++.

  

### 4. Проверить, что найденное `k < rightBorder`

Если да — выводим `YES` и это `k`.
Иначе — `NO`.


## Разбор примера на математике

### Пример

```text
k = 2
парабола: y = x² + x + 5
```

Проверяем пересечения:

```text
x² + x + 5 = 2x
x² - x + 5 = 0
```

Дискриминант:

```text
D = (-1)² - 4 * 1 * 5 = 1 - 20 = -19
```

Он отрицательный, значит пересечений нет, касания тоже нет.
Стратегия подходит.


## Почему работает строгое неравенство

По условию прямая не должна:

* ни пересекать,
* ни касаться параболу.

Если бы было:

```text
D = 0
```

это означало бы касание.

Поэтому нужно не `D <= 0`, а именно:

```text
D < 0
```

Из-за этого и проверка на `k` идёт по **строгому** интервалу:

```text
left < k < right
```

а не с равенствами.

Это явно видно в коде:

* сначала ищется число строго больше левой границы,
* потом проверяется строгое `ans < kx_p`.

 


## Почему можно вывести любой подходящий `k`

По условию, если существует несколько стратегий, можно вывести любую.

Поэтому решение даже не пытается искать “лучшую” стратегию.
Оно просто находит **первое** число в отсортированном массиве, которое попадает в нужный интервал.

Этого достаточно.  


## Сложность

### На один тест

* сортировка `k`: `O(n log n)`
* обработка каждой параболы: `O(log n)`

Итого:

```text
O(n log n + m log n)
```

### По всей задаче

Так как сумма всех `n` и сумма всех `m` по тестам не превосходят `10^5`, решение спокойно укладывается в ограничения.
Код в файлах как раз построен под эту асимптотику.  


## Подводные камни

### 1. Нельзя использовать `<=`

Если `k` попадает ровно на границу интервала, то дискриминант равен нулю, а это касание.
По условию касание запрещено.


### 2. Нужно проверять именно существование числа внутри интервала

Не нужно искать минимальное или максимальное подходящее `k`.
Достаточно найти **любое**.


### 3. Осторожно с типами

Вычисляется:

```text
2 * sqrt(a * c)
```

Поэтому в коде используется `double`.
Это видно и в Python, и в Java, и в C++.   

## Краткий вывод

Вся задача сводится к одному критерию:

Прямая `y = kx` не пересекает и не касается параболы `y = ax² + bx + c` тогда и только тогда, когда

```text
(b - k)² < 4ac
```

то есть

```text
b - 2√(ac) < k < b + 2√(ac)
```

После этого остаётся только:

1. отсортировать все доступные `k`,
2. для каждой параболы бинарным поиском проверить, есть ли `k` внутри нужного интервала.

Именно это и реализовано в предоставленных решениях.
