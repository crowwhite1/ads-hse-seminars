# D. Игра в числа на БИ

## Условие

Дан массив `a` длины `n`.

Нужно построить массив `b` той же длины, где для каждого `i`:

```text
b[i] = a[i] + d[i]
````

где:

* `d[i]` — целое неотрицательное число,
* сумма всех увеличений не превосходит `k`:

```text
d[1] + d[2] + ... + d[n] <= k
```

Требуется минимизировать число `x`, такое что для любых `i != j`:

```text
|b[i] - b[j]| <= x
```

После этого нужно вывести:

1. минимально возможное значение `x`,
2. любой подходящий массив `b`.


# Главная мысль

Условие

```text
|b[i] - b[j]| <= x   для любых i, j
```

эквивалентно тому, что:

```text
max(b) - min(b) <= x
```

То есть нам нужно **сделать диапазон значений массива как можно уже**.

Но есть важное ограничение:
мы умеем только **увеличивать** элементы, уменьшать нельзя.


# Ключевое наблюдение

Обозначим:

```text
M = max(a)
```

Тогда в итоговом массиве:

* максимальный элемент не может стать меньше `M`, потому что уменьшать нельзя,
* если мы увеличим какой-то максимальный элемент, диапазон только ухудшится или не улучшится.

Значит, в оптимальном решении выгодно оставить верхнюю границу равной:

```text
max(b) = M
```

Тогда задача превращается в следующую:

> нужно **максимально поднять минимальный элемент** массива, используя не более `k` увеличений.

Если удастся сделать так, что все элементы будут не меньше некоторого значения `T`, то:

```text
min(b) = T
max(b) = M
x = M - T
```

Значит, минимизация `x` эквивалентна **максимизации `T`**.


# Во что превращается задача

Нужно найти **максимальное** значение `T`, для которого можно поднять все элементы меньше `T` до `T`, потратив не более `k`.

Стоимость такого поднятия:

```text
cost(T) = Σ max(0, T - a[i])
```

Мы хотим найти максимальный `T`, такой что:

```text
cost(T) <= k
```

Тогда оптимальный массив:

```text
b[i] = max(a[i], T)
```

А ответ:

```text
x = M - T
```


# Почему оптимально именно так

## 1. Верхнюю границу нет смысла двигать вверх

Пусть `max(a) = M`.

Так как уменьшать элементы нельзя, в любом массиве `b`:

```text
max(b) >= M
```

Если сделать `max(b) > M`, то диапазон

```text
max(b) - min(b)
```

станет не меньше, чем при `max(b) = M`.

Значит, в оптимальном решении максимальный элемент нужно оставить равным `M`.


## 2. Чтобы уменьшить диапазон, нужно поднимать только маленькие элементы

Если верхняя граница фиксирована и равна `M`, то диапазон минимизируется тогда, когда минимальный элемент максимально возможен.

Следовательно, нужно тратить бюджет на увеличение наименьших элементов.


## 3. В оптимуме существует порог `T`

Если мы хотим, чтобы все элементы были как можно ближе друг к другу, то оптимально выбрать некоторый порог `T` и:

* все элементы меньше `T` поднять до `T`,
* элементы не меньше `T` не трогать.

То есть итоговый массив имеет вид:

```text
b[i] = max(a[i], T)
```

Если какой-то элемент меньше `T`, но мы подняли его не до `T`, а до меньшего значения, это только уменьшает минимум массива и ухудшает ответ.


# Жадная идея

После сортировки массива удобно поднимать элементы **слоями**.

Пусть массив отсортирован:

```text
a[0] <= a[1] <= ... <= a[n-1]
```

Тогда:

* сначала можно поднять `a[0]` до `a[1]`,
* потом `a[0]` и `a[1]` вместе поднять до `a[2]`,
* потом первые 3 элемента поднять до `a[3]`,
* и так далее.

Это очень удобно, потому что стоимость перехода между соседними уровнями считается просто.


# Как считать стоимость поднятия

Допустим, мы уже выровняли первые `j` элементов до уровня `cur`.

Хотим поднять их до следующего значения `a[j]`.

Разница:

```text
delta = a[j] - cur
```

Нужно увеличить каждый из первых `j` элементов на `delta`, значит стоимость:

```text
delta * j
```

Если бюджета хватает — поднимаем.
Если не хватает — дальше уже полностью до следующего уровня дойти нельзя.

Тогда оставшийся бюджет можно равномерно распределить между этими `j` элементами:

```text
extra = remaining / j
```

Итоговый уровень:

```text
T = cur + extra
```


# Полный алгоритм

## Шаг 1

Считываем массив и запоминаем:

```text
M = max(a)
```

## Шаг 2

Сортируем элементы вместе с их исходными индексами.

Это нужно, чтобы потом восстановить массив `b` в исходном порядке.

## Шаг 3

Идем слева направо по отсортированному массиву и пытаемся поднять минимальный префикс до следующего значения.

Если стоимость перехода:

```text
(a[j] - a[j-1]) * j
```

влезает в оставшийся бюджет, делаем переход.

## Шаг 4

Когда полностью подняться до следующего уровня уже нельзя, остаток бюджета равномерно распределяем по текущему префиксу:

```text
extra = (k - spent) / j
T = currentLevel + extra
```

## Шаг 5

Строим ответ:

* первые `j` минимальных элементов становятся равны `T`,
* остальные остаются как были.

## Шаг 6

Выводим:

```text
x = M - T
```

и массив `b`.


# Почему остаток нужно делить именно так

Пусть после полного подъема нескольких слоев у нас осталось:

```text
rem = k - spent
```

Сейчас минимальные `j` элементов уже равны некоторому уровню `cur`.

Чтобы поднять минимум еще на `1`, нужно добавить `1` ко всем этим `j` элементам, то есть потратить `j`.

Значит, максимум, на сколько еще можно поднять минимум:

```text
extra = rem / j
```

Это просто целая часть от деления.


# Разбор на примере

## Пример 1

```text
n = 5, k = 8
a = [1, 2, 7, 3, 10]
```

### Сортировка

```text
[1, 2, 3, 7, 10]
```

### Максимум

```text
M = 10
```

### Поднимаем по слоям

#### Поднять `1` до `2`

Нужно:

```text
(2 - 1) * 1 = 1
```

Бюджет был `8`, осталось `7`.

Теперь первые `1` элементов выровнены до `2`.


#### Поднять первые `2` элемента до `3`

Нужно:

```text
(3 - 2) * 2 = 2
```

Осталось `5`.

Теперь первые `2` элементов выровнены до `3`.


#### Поднять первые `3` элемента до `7`

Нужно:

```text
(7 - 3) * 3 = 12
```

Не хватает.

Значит, полностью до `7` уже не поднимемся.


### Равномерно тратим остаток

Осталось `5`, сейчас `j = 3`, текущий уровень `3`.

Можно поднять все первые `3` элемента еще на:

```text
extra = 5 / 3 = 1
```

Значит:

```text
T = 3 + 1 = 4
```

### Итоговый массив

Первые 3 минимальных элемента становятся `4`:

```text
b = [4, 4, 7, 4, 10]
```

в исходном порядке:

```text
[4, 4, 7, 4, 10]
```

### Ответ

```text
x = 10 - 4 = 6
```


## Пример 2

```text
n = 4, k = 100
a = [5, 5, 5, 5]
```

Все элементы уже равны.
Максимум:

```text
M = 5
```

Минимум тоже `5`, значит:

```text
x = 0
```

Можно вообще ничего не делать.

Итог:

```text
b = [5, 5, 5, 5]
```


## Пример 3

```text
n = 3, k = 2
a = [1, 10, 10]
```

Максимум:

```text
M = 10
```

Чтобы поднять `1` до `10`, нужно `9`, но бюджета всего `2`.

Значит, поднимаем только первый элемент на `2`:

```text
b = [3, 10, 10]
x = 10 - 3 = 7
```


# Корректность алгоритма

## Лемма 1

В оптимальном решении `max(b) = max(a)`.

### Доказательство

Уменьшать элементы нельзя, значит `max(b) >= max(a)`.

Если `max(b) > max(a)`, то диапазон `max(b) - min(b)` не станет меньше, чем при `max(b) = max(a)`, а бюджет будет потрачен зря.

Следовательно, в оптимальном решении выгодно оставить максимум равным `max(a)`. 


## Лемма 2

В оптимальном решении существует число `T`, такое что `b[i] = max(a[i], T)`.

### Доказательство

Пусть в оптимальном решении минимальный элемент равен `T`.

Тогда любой элемент меньше `T` обязан быть увеличен хотя бы до `T`, иначе минимум массива был бы меньше `T`.

Любой элемент больше либо равный `T` нет смысла увеличивать, если мы хотим минимизировать диапазон при фиксированном максимуме.

Следовательно, оптимальный массив имеет вид:

```text
b[i] = max(a[i], T)
```




## Лемма 3

Максимально возможное `T` получается поднятием минимальных элементов по слоям.

### Доказательство

Стоимость поднятия зависит только от того, сколько элементов ниже текущего порога.

В отсортированном массиве все элементы, которые нужно поднимать, образуют префикс.
Поднимать их по соседним уровням — это самый естественный и полный способ проверить, до какого общего уровня можно дойти за бюджет `k`.

Когда до следующего значения полностью дойти уже нельзя, остаток бюджета оптимально делится поровну между всеми текущими минимальными элементами, потому что именно они определяют минимум массива.




## Теорема

Алгоритм находит минимально возможное значение `x` и строит корректный массив `b`.

### Доказательство

По лемме 1 максимум в оптимуме равен `M = max(a)`.
По лемме 2 оптимальный массив имеет вид `b[i] = max(a[i], T)` для некоторого максимального достижимого `T`.
По лемме 3 алгоритм находит именно максимальный достижимый `T`.

Следовательно, он минимизирует:

```text
x = M - T
```

и строит корректный массив `b`. 


# Сложность

## Время

* сортировка: `O(n log n)`
* один проход по массиву: `O(n)`
* восстановление ответа: `O(n)`

Итого:

```text
O(n log n)
```

## Память

* массив значений,
* массив индексов,
* массив ответа.

Итого:

```text
O(n)
```


# Частые ошибки

## 1. Пытаться поднимать какие-то элементы выше максимума

Это никогда не помогает уменьшить диапазон, потому что верхняя граница только растет.


## 2. Использовать весь бюджет обязательно

По условию сумма увеличений должна быть **не больше** `k`, а не ровно `k`.

Если все элементы уже можно сделать равными `max(a)`, то ответ `x = 0`, и остаток бюджета можно просто не использовать.


## 3. Поднимать элементы неравномерно

Если несколько минимальных элементов определяют нижнюю границу массива, то выгодно поднимать именно их вместе.
Иначе минимум останется слишком маленьким.


## 4. Искать ответ бинарным поиском по `x`, когда можно решить жадно

Бинарный поиск тоже возможен, но жадное выравнивание по слоям проще и естественнее для восстановления массива `b`.


# Краткий итог

Задача сводится к очень простой идее:

* максимум массива уменьшить нельзя,
* значит нужно максимально поднять минимум,
* оптимально это делается поднятием наименьших элементов до общего уровня,
* после сортировки это удобно делать по слоям.

Именно поэтому ответ:

```text
x = max(a) - T
```

где `T` — максимальный уровень, до которого можно поднять минимальные элементы за бюджет `k`.
