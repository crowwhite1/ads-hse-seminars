# C. НОП

## Описание задачи

Даны две строки `x` и `y`.

Нужно найти длину их **наибольшей общей подпоследовательности**.

Подпоследовательность получается удалением некоторых символов без изменения порядка оставшихся.  
Символы не обязаны идти подряд.

## Идея решения

Это классическая динамика по двум строкам.

Обозначим:

- `dp[i][j]` — длина НОП для префиксов `x[0..i-1]` и `y[0..j-1]`.

Тогда:

- если `x[i - 1] == y[j - 1]`, этот символ можно добавить к ответу для меньших префиксов:
  `dp[i][j] = dp[i - 1][j - 1] + 1`;
- иначе последний символ хотя бы одной строки не участвует в оптимальном ответе:
  `dp[i][j] = max(dp[i - 1][j], dp[i][j - 1])`.

## Почему это работает

Рассмотрим последние символы текущих префиксов.

Если они совпадают, существует оптимальный ответ, заканчивающийся этим символом, и тогда задача сводится к меньшим префиксам.

Если не совпадают, то хотя бы один из двух последних символов не нужен в оптимальной подпоследовательности.  
Значит нужно попробовать отбросить символ из первой строки или из второй и взять лучший результат.

Это и даёт корректный переход.

## Алгоритм

1. Создать таблицу `dp` размера `(len(x) + 1) x (len(y) + 1)`.
2. Инициализировать нулевую строку и нулевой столбец нулями.
3. Заполнить таблицу по двум переходам:
   - при совпадении символов;
   - при несовпадении символов.
4. Ответ находится в `dp[len(x)][len(y)]`.

## Сложность

Пусть:

- `n = |x|`;
- `m = |y|`.

Тогда:

- время: `O(nm)`;
- память: `O(nm)`.

## Итог

Задача НОП решается стандартной двумерной динамикой по двум строкам, где состояние определяется длинами рассматриваемых префиксов.
