# Наибольшая общая подпоследовательность

Наибольшая общая подпоследовательность двух строк — это самая длинная последовательность символов, которая является подпоследовательностью обеих строк.

Подпоследовательность:

- сохраняет относительный порядок символов;
- не требует, чтобы символы шли подряд.

## Идея алгоритма

Пусть:

- `dp[i][j]` — длина НОП для префиксов длины `i` и `j`.

То есть:

- рассматриваем первые `i` символов строки `x`;
- и первые `j` символов строки `y`.

## Переход

Если последние символы префиксов совпадают:

`dp[i][j] = dp[i - 1][j - 1] + 1`

Если не совпадают:

`dp[i][j] = max(dp[i - 1][j], dp[i][j - 1])`

## Базовые случаи

Если одна из строк пустая, длина общей подпоследовательности равна `0`.

Поэтому:

- `dp[0][j] = 0`;
- `dp[i][0] = 0`.

## Почему это работает

Если последние символы совпадают, один из оптимальных ответов обязательно может оканчиваться этим символом.

Если не совпадают, то хотя бы один из двух последних символов не входит в оптимальную подпоследовательность.  
Значит нужно отбросить либо символ из первой строки, либо из второй, и выбрать лучший из двух вариантов.

Так получаем корректную рекуррентную формулу.

## Наглядный пример

Пусть:

- `x = "ABCBDAB"`
- `y = "BDCABA"`

Одна из общих подпоследовательностей:

- `BCA`

Но можно найти и более длинную:

- `BCBA`

Её длина равна `4`.

Значит для этих строк ответ задачи НОП равен `4`.

Именно это число окажется в ячейке:

- `dp[len(x)][len(y)]`

## Сложность

Пусть:

- `n = |x|`;
- `m = |y|`.

Тогда:

- время: `O(nm)`;
- память: `O(nm)`.

## Итог

НОП — один из самых известных алгоритмов двумерной динамики.

Он показывает, как ответ для двух последовательностей строится по ответам для меньших префиксов.
