# B. Черепашка и монетки

## Описание задачи

Есть таблица `n x m`, в каждой клетке лежит некоторое количество монет.

Черепашка стартует в левом верхнем углу и может двигаться только:

- вправо;
- вниз.

Нужно:

1. найти максимальное количество монет, которое можно собрать по пути в правый нижний угол;
2. восстановить один из маршрутов, на котором достигается этот максимум.

## Идея решения

Это динамика по клеткам таблицы.

Обозначим:

- `dp[i][j]` — максимальное число монет, которое можно собрать, придя в клетку `(i, j)`.

В клетку `(i, j)` можно попасть только:

- сверху из `(i - 1, j)`;
- слева из `(i, j - 1)`.

Значит переход:

`dp[i][j] = max(dp[i - 1][j], dp[i][j - 1]) + coins[i][j]`

## Базовые случаи

- `dp[0][0] = coins[0][0]`
- первую строку можно заполнить только движением вправо;
- первый столбец можно заполнить только движением вниз.

После этого все остальные клетки считаются по общей формуле.

## Восстановление пути

После заполнения таблицы ответ находится в `dp[n - 1][m - 1]`.

Чтобы восстановить маршрут, идём из правого нижнего угла назад:

- если пришли только слева, записываем `R`;
- если только сверху, записываем `D`;
- иначе смотрим, из какой клетки значение `dp` было больше.

Так строится путь в обратном порядке, после чего его нужно развернуть.

## Почему это работает

Для любой клетки оптимальный путь в неё обязательно приходит либо сверху, либо слева.

Если бы лучший путь в `(i, j)` шёл через не лучший путь в предыдущую клетку, его можно было бы улучшить, заменив префикс на оптимальный.

Значит достаточно знать лучший ответ для соседних клеток и взять максимум из них.

Это и есть принцип оптимальности динамического программирования.

## Алгоритм

1. Считать таблицу монет.
2. Заполнить `dp` для первой строки и первого столбца.
3. Для остальных клеток применить переход через максимум из верхней и левой.
4. Вывести `dp[n - 1][m - 1]`.
5. Восстановить путь обратным проходом от конца к началу.

## Сложность

- время: `O(nm)`;
- память: `O(nm)`.

## Итог

Задача решается стандартной двумерной динамикой по сетке с последующим восстановлением ответа.
