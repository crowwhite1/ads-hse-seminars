# Динамическое программирование

Динамическое программирование — это подход, при котором большая задача разбивается на более маленькие подзадачи, а их ответы сохраняются и переиспользуются.

Обычно динамика применяется, когда:

- задача имеет повторяющиеся подзадачи;
- ответ для большого состояния можно выразить через ответы для меньших состояний.

## Из чего состоит динамика

Почти всегда нужно определить три вещи:

1. Что считается состоянием.
2. Какой переход между состояниями.
3. Какие есть базовые случаи.

После этого остаётся выбрать порядок вычисления, чтобы к моменту подсчёта текущего состояния все нужные предыдущие уже были известны.

## Пример 1. Обратный кузнечик

В задаче про кузнечика состоянием удобно взять позицию на лестнице.

- `dp[i]` — число способов добраться из позиции `i` до основания.

Переход:

- из `i` можно прыгнуть в `i + 1`, `i + 2`, `i + 3`;
- значит `dp[i]` равно сумме ответов для этих трёх позиций.

Здесь динамика одномерная:

- состояние задаётся одним числом;
- ответ для текущего состояния зависит от нескольких следующих.

## Пример 2. Черепашка и монетки

В задаче про черепашку состоянием уже является клетка таблицы.

- `dp[i][j]` — максимальное количество монет, которое можно собрать, придя в клетку `(i, j)`.

Переход:

- в клетку можно прийти либо сверху, либо слева;
- значит нужно взять лучший из двух вариантов и добавить монеты текущей клетки.

Это двумерная динамика:

- состояние определяется двумя координатами;
- переход идёт из соседних клеток.

## Что важно заметить на этих двух примерах

Хотя задачи разные, схема решения одна и та же:

- выбираем состояние;
- формулируем переход;
- задаём базу;
- вычисляем значения в правильном порядке.

Разница только в том, что:

- у кузнечика считаем количество способов;
- у черепашки ищем максимум;
- в одной задаче массив одномерный, в другой двумерный.

## Восстановление ответа

Динамика часто позволяет найти не только значение ответа, но и сам ответ.

Например:

- в задаче про черепашку после вычисления таблицы можно пройти из конца назад и восстановить маршрут;
- в других задачах для этого часто хранят массив `prev`, откуда пришли в оптимальное состояние.

## Типичные шаги при решении задач на динамику

1. Понять, что будет состоянием.
2. Определить, что хранит `dp`.
3. Записать переход.
4. Задать базовые значения.
5. Выбрать порядок обхода состояний.
6. При необходимости восстановить ответ.

## Сложность

Чаще всего время динамики равно:

- число состояний
- умножить на стоимость перехода из одного состояния.

Память обычно равна размеру массива `dp`, хотя иногда её можно уменьшить.

## Итог

Динамическое программирование — это не один конкретный алгоритм, а общий способ мыслить о задаче через состояния и переходы.

Первые две задачи семинара хорошо показывают два базовых сценария:

- одномерную динамику по позиции;
- двумерную динамику по таблице.
