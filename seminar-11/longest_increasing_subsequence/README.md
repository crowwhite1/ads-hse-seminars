# Наибольшая возрастающая подпоследовательность

Наибольшая возрастающая подпоследовательность — это подпоследовательность массива, в которой:

- порядок элементов сохраняется;
- каждый следующий элемент строго больше предыдущего;
- длина максимальна среди всех таких подпоследовательностей.

## Что значит подпоследовательность

Подпоследовательность получается удалением некоторых элементов массива без изменения порядка оставшихся.

Элементы не обязаны стоять подряд.

## Идея алгоритма

Пусть:

- `dp[i]` — длина наибольшей возрастающей подпоследовательности, заканчивающейся в позиции `i`;
- `prev[i]` — индекс предыдущего элемента в оптимальной подпоследовательности для позиции `i`.

Тогда для каждого `i` перебираем все `j < i`.

Если `a[j] < a[i]`, то элемент `a[i]` можно поставить после `a[j]`.

Если при этом `dp[j] + 1 > dp[i]`, то найден лучший способ закончить подпоследовательность в `i`.

## Переход

Для всех `j < i`:

```text
if a[j] < a[i]:
    dp[i] = max(dp[i], dp[j] + 1)
```

Одновременно обновляется `prev[i]`, чтобы потом можно было восстановить сам ответ.

## Базовый случай

Каждый элемент сам по себе образует возрастающую подпоследовательность длины `1`.

Поэтому изначально:

- `dp[i] = 1`;
- `prev[i] = -1`.

## Как восстановить ответ

После вычисления всех `dp[i]` выбираем позицию `pos` с максимальным значением `dp[pos]`.

Дальше идём по массиву `prev`:

- берём `a[pos]`;
- переходим в `prev[pos]`;
- продолжаем, пока не получим `-1`.

Подпоследовательность восстанавливается с конца, поэтому её нужно развернуть.

## Почему это работает

Любая возрастающая подпоследовательность, заканчивающаяся в `i`, перед этим заканчивается в некоторой позиции `j < i`, где `a[j] < a[i]`.

Если выбрать среди всех таких позиций ту, которая даёт максимальное значение `dp[j]`, то получим оптимальный ответ для `i`.

Значит динамика корректно находит лучший ответ для каждого конца подпоследовательности, а значит и общий лучший ответ.

## Сложность

Для массива длины `n`:

- время: `O(n^2)`;
- память: `O(n)`.

## Итог

Это классический алгоритм динамического программирования для НВП.

Он прост в реализации и позволяет не только найти длину ответа, но и восстановить саму подпоследовательность.
