# A. Обратный кузнечик

## Описание задачи

Кузнечик сидит на верхней ступеньке лестницы из `n` ступенек и прыгает только вниз.

За один ход он может перейти:

- на следующую ступеньку;
- через одну ступеньку;
- через две ступеньки.

Нужно посчитать количество различных маршрутов спуска с последней ступеньки до основания.  
Ответ требуется вывести по модулю `10^9 + 7`.

## Идея решения

Это классическая динамика по позиции.

Обозначим:

- `dp[i]` — количество способов добраться из позиции `i` до основания.

Тогда из ступеньки `i` можно прыгнуть в:

- `i + 1`;
- `i + 2`;
- `i + 3`.

Значит переход такой:

`dp[i] = dp[i + 1] + dp[i + 2] + dp[i + 3]`

Все вычисления берутся по модулю `10^9 + 7`.

## Базовые случаи

Если смотреть с конца:

- из основания уже есть один способ "добраться до основания" — ничего не делать;
- дальше значения определяются напрямую по допустимым прыжкам.

Именно поэтому массив удобно заполнять справа налево.

## Почему это работает

Любой маршрут из ступеньки `i` начинается с одного из трёх допустимых прыжков.

После первого прыжка остаётся независимо посчитать число способов из новой позиции.

Так как варианты первого прыжка не пересекаются, общее число способов равно сумме количеств способов из достижимых следующих состояний.

Это и даёт формулу динамики.

## Алгоритм

1. Завести массив `dp`.
2. Инициализировать базовые состояния около конца лестницы.
3. Заполнять `dp` справа налево.
4. Ответом будет `dp[0]`, если считать основание позицией `0`.

## Сложность

- время: `O(n)`;
- память: `O(n)`.

## Итог

Задача решается одномерной динамикой, где состояние — текущая ступенька, а переходы соответствуют трём возможным прыжкам вниз.
