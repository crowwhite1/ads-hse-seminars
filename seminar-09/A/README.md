# Соединение графа с помощью DSU (Union-Find)

## Описание задачи

Дан набор вершин (островов) и последовательность рёбер (порталов), которые добавляются по очереди.

Каждое ребро соединяет две вершины.  
Гарантируется, что рёбра добавляются в фиксированном порядке.

Необходимо определить минимальное количество первых рёбер, после добавления которых граф становится связным.

## Идея решения

Задача решается с помощью структуры данных **DSU (Disjoint Set Union)**, также известной как **Union-Find**.

DSU позволяет эффективно:

- определять, находятся ли две вершины в одной компоненте связности,
- объединять компоненты.

## Основная идея

Изначально каждая вершина находится в своей компоненте.

Далее мы последовательно добавляем рёбра:

- если вершины уже находятся в одной компоненте — ничего не меняется,
- если в разных — объединяем их.

Нам важно найти момент, когда все вершины окажутся в одной компоненте.

В данном решении отслеживается индекс последнего "полезного" ребра — того, которое действительно объединяет две разные компоненты.

## Как работает алгоритм

### 1. Инициализация

Каждая вершина является своим родителем:

- `parent[v] = v`
- размер компоненты равен 1

### 2. Обработка рёбер

Для каждого ребра `(u, v)`:

- проверяем, лежат ли `u` и `v` в одной компоненте,
- если нет — объединяем их,
- запоминаем индекс этого ребра как последний полезный.

### 3. Ответ

Ответом является количество рёбер, которые реально понадобились для соединения графа.

Это равно:

`last_index + 1`

## Почему это работает

Граф из `n` вершин становится связным, когда в нём появляется ровно `n - 1` "полезных" рёбер, соединяющих разные компоненты.

DSU гарантирует:

- каждое объединение уменьшает число компонент на 1,
- лишние рёбра (внутри компоненты) игнорируются.

Таким образом, последнее объединение происходит ровно в тот момент, когда граф становится связным.

## Структура DSU

### Операции

- `get(x)` — найти представителя множества (сжатие пути),
- `union(a, b)` — объединить множества,
- `is_same(a, b)` — проверить, в одной ли компоненте.

### Оптимизации

- **Сжатие пути (path compression)** — ускоряет поиск корня,
- **Объединение по размеру (union by size)** — уменьшает высоту дерева.

## Сложность

### По времени

Практически:

`O(m α(n))`

где `α(n)` — обратная функция Аккермана (очень медленно растёт, можно считать константой).

### По памяти

`O(n)`

## Особенности

- граф не обязательно связный изначально;
- рёбра могут повторяться;
- порядок рёбер важен — нельзя менять их местами;
- решение работает онлайн (обрабатывает рёбра по мере поступления).

## Итог

DSU — оптимальный инструмент для задач на динамическое объединение компонент.

Он позволяет за почти линейное время определить момент, когда граф становится связным.
