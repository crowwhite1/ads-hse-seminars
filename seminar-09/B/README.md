# Минимальное остовное дерево (Kruskal + DSU)

## Описание задачи

Дан неориентированный взвешенный граф.  
Требуется найти вес минимального остовного дерева (MST).

Минимальное остовное дерево — это подмножество рёбер, которое:

- соединяет все вершины графа,
- не содержит циклов,
- имеет минимальный возможный суммарный вес.

## Идея решения

Решение основано на алгоритме **Краскала (Kruskal)**.

Алгоритм строит остовное дерево, последовательно добавляя рёбра с наименьшим весом, избегая образования циклов.

Для проверки наличия циклов используется структура **DSU (Disjoint Set Union)**.

## Алгоритм Краскала

### Шаги

1. Считать все рёбра графа
2. Отсортировать рёбра по возрастанию веса
3. Инициализировать DSU (каждая вершина — отдельная компонента)
4. Перебирать рёбра в порядке возрастания:
   - если вершины ребра находятся в разных компонентах:
     - добавить ребро в остов
     - объединить компоненты
     - увеличить суммарный вес
   - иначе пропустить (иначе возникнет цикл)

## Почему это работает

Алгоритм Краскала основан на жадном принципе:

- на каждом шаге выбирается минимальное возможное ребро,
- которое не нарушает структуру дерева (не создаёт цикл).

Теорема о корректности гарантирует, что такой подход всегда приводит к минимальному остовному дереву.

## Роль DSU

DSU используется для:

- проверки, находятся ли две вершины в одной компоненте,
- объединения компонент.

Это позволяет эффективно избегать циклов.

## Структура DSU

### Операции

- `get(x)` — найти представителя множества
- `union(a, b)` — объединить множества
- `is_same(a, b)` — проверить, соединены ли вершины

### Оптимизации

- **Сжатие пути (path compression)**
- **Объединение по размеру (union by size)**

Обе оптимизации делают операции почти константными.

## Сложность

### По времени

- сортировка рёбер: `O(m log m)`
- операции DSU: `O(m α(n))`

Итог:

`O(m log m)`

### По памяти

`O(n + m)`

## Особенности

- граф должен быть связным (иначе получится остовный лес);
- рёбра могут быть в любом порядке;
- веса рёбер могут повторяться;
- используется жадный алгоритм.

## Итог

Алгоритм Краскала + DSU — стандартный и эффективный способ нахождения минимального остовного дерева.

Он прост в реализации и хорошо работает на больших графах.
