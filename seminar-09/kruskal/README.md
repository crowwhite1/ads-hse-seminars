# Алгоритм Краскала

Алгоритм Краскала используется для построения минимального остовного дерева в неориентированном взвешенном графе.

Минимальное остовное дерево — это набор рёбер, который:

- соединяет все вершины;
- не содержит циклов;
- имеет минимальный суммарный вес.

## Идея алгоритма

Алгоритм действует жадно:

- сортирует все рёбра по возрастанию веса;
- идёт по ним слева направо;
- добавляет ребро, только если оно соединяет разные компоненты связности.

Если ребро соединяет вершины внутри одной компоненты, его добавление образовало бы цикл, поэтому такое ребро пропускается.

## Как определять компоненты

Для быстрой проверки обычно используется DSU.

DSU позволяет:

- узнать, находятся ли две вершины в одной компоненте;
- объединить две компоненты после добавления ребра.

## Шаги алгоритма

1. Считать все рёбра графа.
2. Отсортировать их по весу.
3. Инициализировать DSU, где каждая вершина — отдельная компонента.
4. Перебирать рёбра в порядке возрастания веса:
   - если концы ребра в разных компонентах, добавить ребро в остов и объединить компоненты;
   - иначе пропустить ребро.

## Почему алгоритм корректен

Краскал опирается на жадный выбор:

- на каждом шаге берётся минимальное по весу ребро,
- которое не нарушает ацикличность строящегося остова.

Теория минимальных остовов гарантирует, что такой выбор безопасен и в итоге приводит к минимальному остовному дереву.

## Что получается на выходе

- если граф связный, получится минимальное остовное дерево;
- если граф несвязный, получится минимальный остовный лес.

## Сложность

Основное время тратится на сортировку:

`O(m log m)`

где:

- `n` — число вершин;
- `m` — число рёбер.

Операции DSU добавляют:

`O(m α(n))`

Итоговая сложность:

`O(m log m)`

Память:

`O(n + m)`

## Особенности

- алгоритм работает для неориентированного графа;
- веса рёбер могут повторяться;
- порядок рёбер до сортировки не важен;
- для эффективной реализации почти всегда используют DSU.

## Итог

Алгоритм Краскала — один из самых стандартных способов строить минимальное остовное дерево.

Он прост в реализации, хорошо сочетается с DSU и эффективен на больших графах.
