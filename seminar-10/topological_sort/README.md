# Топологическая сортировка

Топологическая сортировка — это упорядочивание вершин ориентированного графа, при котором для каждого ребра `u -> v` вершина `u` стоит раньше вершины `v`.

Такой порядок существует только для ацикличного ориентированного графа, то есть для DAG.

## Когда используется

Топологическая сортировка нужна в задачах, где есть зависимости:

- порядок выполнения задач;
- порядок сборки модулей;
- курсы и их пререквизиты;
- вычисления по зависимостям;
- любые частичные порядки.

## Идея алгоритма

Один из стандартных способов построить топологический порядок — DFS.

Во время обхода:

- сначала рекурсивно обрабатываются все вершины, в которые есть исходящие рёбра;
- затем текущая вершина добавляется в список ответа.

После завершения DFS список разворачивается.

## Почему это работает

Если есть ребро `u -> v`, то при обходе:

- либо вершина `v` будет полностью обработана раньше `u`,
- либо она уже была обработана раньше в другом DFS.

В обоих случаях `u` добавится в список не раньше `v`, а после разворота окажется раньше `v`.

Поэтому полученный порядок удовлетворяет всем рёбрам графа.

## Как обнаружить цикл

Для этого удобно использовать раскраску вершин:

- `0` — вершина не посещена;
- `1` — вершина находится в стеке рекурсии;
- `2` — вершина полностью обработана.

Если из вершины `v` есть ребро в вершину цвета `1`, найден цикл.

Тогда топологической сортировки не существует.

## Алгоритм

1. Завести массив цветов.
2. Запустить DFS из всех непосещённых вершин.
3. При выходе из вершины добавлять её в список `order`.
4. Если найдено ребро в вершину цвета `1`, остановиться: граф цикличен.
5. Иначе развернуть `order`.

## Сложность

Пусть:

- `n` — число вершин;
- `m` — число рёбер.

Тогда:

- время: `O(n + m)`;
- память: `O(n + m)`.

## Итог

Топологическая сортировка — базовый алгоритм для DAG.

DFS позволяет за линейное время либо построить корректный порядок вершин, либо обнаружить, что из-за цикла такой порядок не существует.
