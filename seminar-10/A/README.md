# A. Производство деталей

## Описание задачи

Есть `n` деталей. Для каждой детали известно:

- время её изготовления `p[i]`;
- список деталей, которые нужно изготовить заранее.

Предприятие может одновременно производить только одну деталь, поэтому общее время равно сумме времён всех деталей, которые реально понадобятся для изготовления детали `1`.

Нужно:

1. определить минимальное время, за которое можно изготовить деталь `1`;
2. вывести порядок изготовления всех необходимых деталей.

Гарантируется, что циклических зависимостей нет.

## Идея решения

Нас интересуют не все детали, а только те, от которых зависит деталь `1`.

Если рассматривать зависимости как ориентированный граф:

- вершина — это деталь;
- ребро `u -> v` означает, что для изготовления `u` нужна деталь `v`.

Тогда задача сводится к обходу всех вершин, достижимых из вершины `1`.

Используется DFS из вершины `1`:

- сначала рекурсивно обрабатываем все зависимости;
- потом добавляем текущую деталь в ответ.

Такой порядок является топологическим для подграфа нужных деталей: каждая зависимость попадает в ответ раньше детали, которой она нужна.

## Почему время минимально

Чтобы изготовить деталь `1`, необходимо изготовить все детали, от которых она зависит напрямую или косвенно.

Никакую из этих деталей убрать нельзя, иначе деталь `1` не получится произвести.

Также не нужно изготавливать детали, которые не достижимы из вершины `1`, потому что они никак не влияют на её производство.

Значит минимальное время — это просто сумма времён всех достижимых из `1` деталей.

## Почему порядок корректен

DFS добавляет вершину в ответ после обработки всех её потомков.

Следовательно:

- если деталь `u` зависит от детали `v`,
- то `v` окажется в ответе раньше `u`.

Именно это и требуется для корректного порядка производства.

## Алгоритм

1. Считать времена изготовления.
2. Построить граф зависимостей.
3. Запустить DFS из вершины `1`.
4. Во время выхода из рекурсии добавлять вершины в список `order`.
5. Суммировать стоимости деталей из `order`.
6. Вывести сумму, количество деталей и сам порядок.

## Сложность

Пусть:

- `n` — число деталей;
- `m` — суммарное число зависимостей.

Тогда:

- время: `O(n + m)` в худшем случае;
- память: `O(n + m)`.

## Итог

Задача решается обычным DFS по графу зависимостей.

Обход позволяет:

- выбрать только действительно нужные детали;
- получить корректный порядок их изготовления;
- посчитать минимальное суммарное время производства детали `1`.
