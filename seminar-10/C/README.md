# C. Конденсация

## Описание задачи

Дан ориентированный граф. Требуется:

1. найти все компоненты сильной связности;
2. пронумеровать их так, чтобы для любого ребра номер компоненты начала не превышал номер компоненты конца.

Иными словами, нужно построить конденсацию графа и выдать номера компонент в топологическом порядке.

## Что такое компонента сильной связности

Компонента сильной связности — это максимальное множество вершин, в котором из любой вершины можно добраться до любой другой.

Если каждую такую компоненту сжать в одну вершину, получится ацикличный граф, который называется конденсацией.

## Идея решения

Используется алгоритм Косарайю.

Он состоит из двух обходов DFS:

### 1. Первый проход по исходному графу

Запускаем DFS и записываем вершины в список по времени выхода.

Чем позже вершина завершилась, тем раньше она должна рассматриваться на втором проходе.

### 2. Второй проход по транспонированному графу

Строим граф с обращёнными рёбрами.

Далее обрабатываем вершины в порядке убывания времени выхода из первого прохода.

Каждый запуск DFS на обратном графе выделяет ровно одну компоненту сильной связности.

## Почему нумерация получается правильной

Если во втором проходе идти в порядке убывания времени выхода, то компоненты будут находиться в топологическом порядке конденсации.

Поэтому, если компонентам присваивать номера в порядке их нахождения:

- для любого ребра между компонентами
- номер компоненты-источника не будет больше номера компоненты-приёмника.

Именно это и требуется в условии.

## Алгоритм

1. Считать граф и одновременно построить обратный граф.
2. Выполнить DFS по исходному графу и сохранить порядок выхода вершин.
3. Развернуть этот порядок.
4. Запустить DFS по обратному графу в полученном порядке.
5. Каждому новому запуску присваивать следующий номер компоненты.
6. Вывести число компонент и номер компоненты для каждой вершины.

## Сложность

Пусть:

- `n` — число вершин;
- `m` — число рёбер.

Тогда:

- время: `O(n + m)`;
- память: `O(n + m)`.

## Итог

Алгоритм Косарайю позволяет за линейное время найти все компоненты сильной связности и сразу получить их корректную нумерацию в порядке, совместимом с конденсацией графа.
