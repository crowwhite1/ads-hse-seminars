# Алгоритм Косарайю

Алгоритм Косарайю используется для поиска компонент сильной связности в ориентированном графе.

Компонента сильной связности — это максимальное множество вершин, в котором из любой вершины можно добраться до любой другой.

## Что делает алгоритм

Алгоритм разбивает ориентированный граф на компоненты сильной связности.

После этого можно:

- сжать каждую компоненту в одну вершину;
- получить конденсацию графа;
- работать уже с ацикличным графом компонент.

## Идея алгоритма

Алгоритм состоит из двух проходов DFS.

### Первый проход

DFS запускается по исходному графу.

Вершина добавляется в список `order` после обработки всех исходящих рёбер.

Так формируется порядок по времени выхода из DFS.

### Второй проход

Строится транспонированный граф, то есть граф со всеми рёбрами в обратную сторону.

Далее вершины рассматриваются в порядке, обратном порядку выхода из первого прохода.

Каждый запуск DFS на обратном графе выделяет одну компоненту сильной связности.

## Почему это работает

Если рассмотреть конденсацию графа, то это DAG.

Первый DFS фактически расставляет компоненты так, что на втором проходе мы начинаем обход с таких вершин, из которых в обратном графе можно полностью выделить ровно одну компоненту, не захватывая чужие.

Поэтому каждый новый запуск второго DFS находит ровно одну КСС.

## Алгоритм

1. Построить исходный граф.
2. Построить обратный граф.
3. Запустить DFS по исходному графу и записать вершины в `order` по выходу.
4. Развернуть `order`.
5. Запускать DFS по обратному графу в этом порядке.
6. Всем вершинам, посещённым в одном таком DFS, присвоить один номер компоненты.

## Что возвращает реализация

Обычно алгоритм возвращает массив:

- `component[v]` — номер компоненты сильной связности вершины `v`.

При необходимости отдельно можно хранить и количество компонент.

## Сложность

Пусть:

- `n` — число вершин;
- `m` — число рёбер.

Тогда:

- время: `O(n + m)`;
- память: `O(n + m)`.

## Особенности

- алгоритм работает только для ориентированных графов;
- нужен и исходный, и обратный граф;
- конденсация графа после сжатия компонент всегда ациклична.

## Итог

Алгоритм Косарайю — классический линейный алгоритм для поиска компонент сильной связности.

Он прост в реализации и особенно удобен, когда список смежности и обратный граф легко построить заранее.
