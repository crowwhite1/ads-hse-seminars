# A. Мосты

## Описание задачи

Дан неориентированный граф. Требуется найти все мосты.

Мостом называется ребро, удаление которого увеличивает количество компонент связности графа.  
Иными словами, если после удаления ребра из одной вершины в другую уже нельзя добраться так же, как раньше, то это ребро является мостом.

Граф может быть несвязным, поэтому решение должно корректно обрабатывать несколько компонент связности.

## Идея решения

Для поиска мостов используется обход графа в глубину (DFS) и классический алгоритм на массивах `tin` и `low`.

Во время обхода для каждой вершины запоминаются:

- `tin[v]` — время входа в вершину `v`,
- `low[v]` — минимальное время входа, достижимое из вершины `v` по:
  - одному или нескольким рёбрам дерева DFS,
  - и не более чем одному обратному ребру.

### Что это даёт

Если из поддерева вершины `to` нельзя подняться обратно к вершине `v` или выше неё, то ребро `(v, to)` является мостом.

Это проверяется условием:

`low[to] > tin[v]`

Если оно выполняется, значит из поддерева `to` нет пути в предков `v`, кроме как через ребро `(v, to)`. Следовательно, это ребро — мост.

## Как работает алгоритм

### 1. Построение графа

Граф хранится в виде списка смежности.  
Для каждой вершины сохраняется список всех соседей.

Одновременно при чтении входных данных запоминается:

- номер каждого ребра во входе,
- информация о кратных рёбрах.

Это нужно потому, что в графе могут быть несколько рёбер между одной и той же парой вершин.  
Такие рёбра не могут считаться мостами в обычном смысле: если удалить одно из них, связь между вершинами всё равно останется через другое.

### 2. DFS-обход

Для каждой ещё не посещённой вершины запускается DFS, потому что граф может состоять из нескольких компонент.

Во время обхода:

- вершина помечается как посещённая,
- ей присваивается время входа,
- далее перебираются все её соседи.

#### Возможны два случая:

**Сосед уже посещён**  
Это означает наличие обратного ребра. Тогда обновляется значение `low`.

**Сосед ещё не посещён**  
Переходим в него рекурсивно, а после возвращения обновляем `low` текущей вершины через `low` потомка.

После этого проверяется условие моста.

### 3. Обработка кратных рёбер

После того как все потенциальные мосты найдены через DFS, они дополнительно фильтруются.

Если ребро относится к кратным, оно не добавляется в ответ.  
В итоговый список попадают только настоящие мосты, после чего номера рёбер сортируются по возрастанию.

## Почему решение корректно

Алгоритм DFS с массивами `tin` и `low` корректно находит все мосты в неориентированном графе:

- `tin[v]` показывает момент первого посещения вершины,
- `low[v]` показывает, можно ли из поддерева вершины добраться назад к предкам.

Если для ребра из `v` в `to` выполняется `low[to] > tin[v]`, то поддерево `to` не имеет альтернативного пути к вершине `v` или выше.  
Значит удаление этого ребра разрывает связность, следовательно, это мост.

Отдельная фильтрация кратных рёбер необходима, потому что наличие параллельного ребра сохраняет связь между вершинами даже после удаления одного из них.

Таким образом, алгоритм находит все и только те рёбра, которые действительно являются мостами.

## Оценка сложности

### По времени

- построение графа — `O(m)`,
- DFS — `O(n + m)`,
- сортировка ответа — `O(b log b)`, где `b` — количество мостов.

Итоговая сложность:

`O(n + m + b log b)`

На практике основная часть работы — это линейный обход графа.

### По памяти

Используются:

- список смежности,
- массивы посещения,
- массивы `tin` и `low`,
- структуры для хранения номеров рёбер и кратных рёбер.

Итоговая сложность по памяти:

`O(n + m)`

## Особенности решения

- граф может быть несвязным;
- рёбра нумеруются в порядке ввода;
- ответ выводится в порядке возрастания номеров рёбер;
- кратные рёбра не считаются мостами;
- решение подходит под ограничения задачи.

## Итог

Решение основано на классическом алгоритме поиска мостов через DFS и значения `tin/low`.  
Оно эффективно работает даже на больших графах и корректно учитывает важную особенность задачи — наличие кратных рёбер.
