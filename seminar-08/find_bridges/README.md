# Поиск мостов в неориентированном графе

## Что такое мост

Мост — это ребро неориентированного графа, удаление которого увеличивает количество компонент связности.

Проще говоря, если удалить такое ребро, то некоторые вершины больше не смогут добраться друг до друга.

## Идея алгоритма

Для поиска мостов используется обход в глубину, DFS.

Во время обхода для каждой вершины вычисляются два значения:

- `tin[v]` — время входа в вершину `v`,
- `low[v]` — минимальное время входа, достижимое из вершины `v`:
  - либо по рёбрам дерева DFS,
  - либо по одному обратному ребру.

## Что означает `low[v]`

Значение `low[v]` показывает, насколько высоко по дереву обхода можно подняться из поддерева вершины `v`.

Если из поддерева некоторой вершины `to` нельзя попасть обратно в вершину `v` или выше, значит ребро между `v` и `to` критично.

## Условие моста

Пусть в DFS из вершины `v` мы пошли в вершину `to`.

После обработки `to` проверяется условие:

`low[to] > tin[v]`

Если оно выполняется, то ребро `(v, to)` является мостом.

### Почему

Это означает, что из поддерева `to` нет пути назад к `v` или к её предкам.

Следовательно, единственный способ связаться с остальной частью графа проходит через ребро `(v, to)`.  
Если удалить это ребро, граф распадётся на части.

## Как работает алгоритм

### 1. Запускаем DFS

Обход выполняется из каждой непосещённой вершины, потому что граф может быть несвязным.

### 2. Запоминаем время входа

При первом посещении вершины `v`:

- отмечаем её как посещённую,
- записываем `tin[v]`,
- изначально полагаем `low[v] = tin[v]`.

### 3. Обрабатываем соседей

Для каждого соседа `to`:

#### Если `to` — родитель в DFS-дереве

Такое ребро пропускается.

#### Если `to` уже посещена

Это обратное ребро. Тогда:

`low[v] = min(low[v], tin[to])`

#### Если `to` ещё не посещена

Идём в неё рекурсивно, а после возврата обновляем:

`low[v] = min(low[v], low[to])`

После этого проверяем, является ли ребро мостом.

## Корректность

Алгоритм корректен, потому что:

- `tin[v]` хранит порядок входа в вершины,
- `low[v]` показывает, можно ли из поддерева добраться обратно вверх по DFS-дереву.

Если для ребра `(v, to)` выполняется `low[to] > tin[v]`, то поддерево `to` не имеет обходного пути назад.  
Значит это ребро обязательно для связности, то есть это мост.

Если условие не выполняется, существует альтернативный путь, и удаление ребра не разрывает граф.

## Сложность

### По времени

DFS посещает каждую вершину один раз и каждое ребро рассматривает константное число раз.

Итоговая сложность:

`O(n + m)`

где:

- `n` — число вершин,
- `m` — число рёбер.

### По памяти

Используются:

- список смежности,
- массивы `visited`, `tin`, `low`,
- список найденных мостов.

Итоговая сложность по памяти:

`O(n + m)`

## Особенности

- Алгоритм работает только для неориентированного графа.
- Если граф несвязный, DFS нужно запускать из каждой непосещённой вершины.
- В базовой форме алгоритм предполагает аккуратную обработку кратных рёбер, если они разрешены в постановке.

## Итог

Поиск мостов через DFS и массивы `tin`/`low` — это классический линейный алгоритм для неориентированного графа.

Он позволяет эффективно определить все рёбра, удаление которых нарушает связность графа.
