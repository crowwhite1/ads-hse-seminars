# Поиск точек сочленения в неориентированном графе

## Что такое точка сочленения

Точка сочленения — это вершина неориентированного графа, удаление которой увеличивает количество компонент связности.

Проще говоря, если убрать такую вершину вместе со всеми её рёбрами, то граф распадётся на большее число частей.

## Идея алгоритма

Для поиска точек сочленения используется обход в глубину, DFS.

Во время обхода для каждой вершины вычисляются два значения:

- `tin[v]` — время входа в вершину `v`,
- `low[v]` — минимальное время входа, достижимое из вершины `v`:
  - по рёбрам дерева DFS,
  - или по одному обратному ребру.

## Что означает `low[v]`

Значение `low[v]` показывает, можно ли из поддерева вершины `v` добраться обратно к самой вершине `v` или к её предкам в DFS-дереве.

Это позволяет понять, зависит ли некоторое поддерево от текущей вершины с точки зрения связности.

## Когда вершина является точкой сочленения

Пусть во время DFS из вершины `v` мы переходим в вершину `to`.

После обработки `to` возможны два случая.

### 1. Вершина `v` не является корнем DFS

Если выполняется условие:

`low[to] >= tin[v]`

то вершина `v` является точкой сочленения.

### Почему

Это означает, что из поддерева `to` нельзя попасть в предков вершины `v`, минуя саму `v`.

Следовательно, если удалить `v`, поддерево `to` потеряет связь с остальной частью графа.

### 2. Вершина `v` является корнем DFS

Для корня действует отдельное правило:

если у корня два или более детей в дереве DFS, то он является точкой сочленения.

### Почему

Каждое такое поддерево связано с остальным графом только через корень.  
Если удалить корень, эти части окажутся разъединены.

## Как работает алгоритм

### 1. Запускаем DFS

Граф может быть несвязным, поэтому обход запускается из каждой непосещённой вершины.

### 2. Запоминаем время входа

При первом посещении вершины `v`:

- помечаем её как посещённую,
- записываем `tin[v]`,
- изначально задаём `low[v] = tin[v]`.

### 3. Обрабатываем соседей

Для каждого соседа `to`:

#### Если `to` — родитель

Такое ребро пропускается.

#### Если `to` уже посещена

Это обратное ребро. Тогда:

`low[v] = min(low[v], tin[to])`

#### Если `to` ещё не посещена

Переходим в неё рекурсивно, а после возврата:

- обновляем `low[v]`,
- проверяем условие точки сочленения,
- увеличиваем число детей в DFS-дереве.

## Корректность

Алгоритм корректен, потому что:

- `tin[v]` хранит порядок обхода,
- `low[v]` показывает, можно ли из поддерева подняться вверх по дереву DFS без использования текущей вершины как обязательной точки прохода.

Если для ребёнка `to` выполняется `low[to] >= tin[v]`, значит поддерево `to` не имеет альтернативного пути к предкам `v`.  
Тогда вершина `v` критична для связности и является точкой сочленения.

Для корня используется отдельное правило, потому что у него нет предков, и его роль определяется числом независимых поддеревьев.

## Сложность

### По времени

DFS посещает каждую вершину один раз и рассматривает каждое ребро константное число раз.

Итоговая сложность:

`O(n + m)`

где:

- `n` — число вершин,
- `m` — число рёбер.

### По памяти

Используются:

- список смежности,
- массивы `visited`, `tin`, `low`,
- множество или список для хранения ответа.

Итоговая сложность по памяти:

`O(n + m)`

## Особенности

- Алгоритм работает для неориентированного графа.
- Если граф несвязный, DFS нужно запускать из каждой непосещённой вершины.
- Одна и та же вершина может подходить под условие несколько раз, поэтому удобно хранить ответ в множестве.

## Итог

Поиск точек сочленения через DFS и массивы `tin`/`low` — это классический линейный алгоритм для неориентированного графа.

Он позволяет эффективно найти все вершины, удаление которых нарушает связность графа.
