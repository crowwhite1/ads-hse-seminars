# Точки сочленения (Articulation Points)

## Описание задачи

Дан неориентированный граф. Требуется найти все точки сочленения.

Точка сочленения (или articulation point) — это вершина, удаление которой увеличивает количество компонент связности графа.  
Иначе говоря, если убрать вершину (вместе со всеми её рёбрами), и граф станет менее связным, то эта вершина является точкой сочленения.

Граф может быть несвязным, поэтому необходимо обработать все его компоненты.

## Идея решения

Решение основано на обходе графа в глубину (DFS) с использованием двух массивов:

- `tin[v]` — время входа в вершину `v`,
- `low[v]` — минимальное время входа, достижимое из вершины `v`.

### Смысл `low[v]`

`low[v]` показывает, можно ли из поддерева вершины `v` добраться обратно:

- к самой вершине `v`,
- или к её предкам в DFS-дереве.

## Ключевая идея

Вершина `v` является точкой сочленения, если выполняется одно из условий:

### 1. Обычная вершина (не корень DFS)

Если существует сын `to`, для которого:

`low[to] >= tin[v]`

Это означает, что поддерево `to` не имеет обратного пути к предкам `v`.  
Следовательно, удаление `v` разрывает связь между этим поддеревом и остальной частью графа.

### 2. Корень DFS

Если вершина является корнем DFS (то есть у неё нет родителя), то она является точкой сочленения, если у неё **2 или более детей** в DFS-дереве.

Это значит, что изначально граф соединял эти поддеревья только через корень.

## Как работает алгоритм

### 1. Построение графа

Граф хранится в виде списка смежности.  
Для каждой вершины хранится список её соседей.

### 2. DFS-обход

Для каждой непосещённой вершины запускается DFS (граф может быть несвязным).

Во время обхода:

- фиксируется время входа (`tin`),
- вычисляется `low`,
- считается количество детей вершины в DFS-дереве.

### 3. Обработка рёбер

При обходе соседей возможны два случая:

#### Уже посещённая вершина

Это обратное ребро.  
Обновляем:

`low[v] = min(low[v], tin[to])`

#### Непосещённая вершина

Переходим в неё рекурсивно:

- увеличиваем счётчик детей,
- после возврата обновляем `low[v]`,
- проверяем условие точки сочленения.

### 4. Формирование ответа

Все найденные вершины сохраняются в множество (чтобы избежать повторов).

После завершения обхода:

- вершины переводятся в 1-based нумерацию,
- сортируются по возрастанию,
- выводятся.

## Почему решение корректно

Алгоритм корректен благодаря свойствам DFS:

- `tin[v]` фиксирует порядок обхода,
- `low[v]` показывает, есть ли альтернативный путь вверх по дереву.

Если для потомка `to` выполняется `low[to] >= tin[v]`, значит:

- поддерево `to` не может обойти вершину `v`,
- удаление `v` разрывает граф.

Отдельное условие для корня необходимо, потому что у него нет предков, и его роль в связности определяется числом независимых поддеревьев.

## Оценка сложности

### По времени

- построение графа: `O(m)`
- DFS: `O(n + m)`
- сортировка ответа: `O(k log k)`, где `k` — число точек сочленения

Итог:

`O(n + m)`

### По памяти

Используются:

- список смежности,
- массивы `visited`, `tin`, `low`,
- множество для хранения ответа.

Итог:

`O(n + m)`

## Особенности решения

- граф может быть несвязным;
- вершины выводятся в порядке возрастания;
- используется множество для исключения дубликатов;
- решение эффективно работает при больших ограничениях.

## Итог

Алгоритм использует DFS и значения `tin/low` для определения критических вершин графа.  
Он позволяет за линейное время найти все точки сочленения даже в больших графах.
