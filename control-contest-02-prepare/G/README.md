# Задача G. Отбор на военку

## 📌 Условие

Дан ориентированный граф:
- вершины — пункты (от `1` до `N`)
- рёбра — тропы с весами (веса могут быть **отрицательными**)

Нужно пройти путь:
```

1 → N

```

и максимизировать сумму весов.


## 🎯 Задача

Найти:
- максимальную сумму весов на пути из `1` в `N`

Возможны три случая:
- есть путь → вывести максимум
- пути нет → `"Impossible"`
- можно увеличить бесконечно → `"Infinity"`


## 🔢 Формат входных данных

- `N (1 ≤ N ≤ 2000)` — вершины  
- `M (1 ≤ M ≤ 10000)` — рёбра  
- далее `M` троек `(a, b, w)`


## 📤 Формат выходных данных

- максимальная сумма
- или `"Impossible"`
- или `"Infinity"`


## 💡 Идея задачи

Это задача на:
> **поиск максимального пути в графе с возможными циклами**

⚠️ Но:
- веса могут быть отрицательные
- есть циклы

👉 значит:
> обычные алгоритмы (DFS / Dijkstra) не подходят


## 🧠 Ключевая идея

Используем:
> **Bellman-Ford (на максимум)**


## ⚙️ Шаг 1. Максимальные расстояния

Инициализация:
```

dist[1] = 0
dist[остальные] = -∞

```

Далее `N-1` раз:
```

dist[v] = max(dist[v], dist[u] + w)

```


## ⚙️ Шаг 2. Проверка достижимости

Если:
```

dist[N] = -∞

```

👉 пути нет → `"Impossible"`


## ⚙️ Шаг 3. Поиск "плохих" циклов

Если после `N-1` итераций можно ещё улучшить:
```

dist[v] < dist[u] + w

```

👉 есть **положительный цикл**


## ⚙️ Шаг 4. Важно!

Цикл должен:
- быть достижим из `1`
- из него можно дойти до `N`

👉 иначе он не влияет на ответ


## ⚙️ Как проверить

1. BFS/DFS от `1` → reachable  
2. BFS/DFS от `N` по обратному графу → canReach  

Если вершина:
```

reachable[v] == True
и canReach[v] == True

```

👉 цикл влияет → `"Infinity"`


## 🧩 Почему это работает

- Bellman-Ford находит максимум
- дополнительная итерация выявляет циклы
- фильтрация оставляет только полезные циклы


## ⏱ Сложность

- Bellman-Ford: **O(N * M)**
- BFS: **O(N + M)**

👉 итог: **O(N * M)**


## 📘 Пример

### Вход:
```

8 7
1 8 1
6 4 -10
3 8 -10
1 7 8
3 4 -2
7 8 1
5 3 -7

```

### Выход:
```

9

```


## 🚀 Важные моменты

- ищем **максимум**, а не минимум
- используем `-∞`, а не `+∞`
- обязательно проверять циклы
- учитывать достижимость


## 🎯 Зачем эта задача

- Bellman-Ford (продвинутый уровень)
- работа с циклами
- фильтрация полезных путей
