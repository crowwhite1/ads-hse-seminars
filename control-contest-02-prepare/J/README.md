# Задача J. Банки и кабели

## 📌 Условие

Дан ориентированный граф:

- вершины — дома
- рёбра — интернет-кабели, по которым можно переходить из одного дома в другой
- у каждого дома `i` есть значение `a[i]` — размер банки с огурцами

Вова хочет пройти маршрут длиной `k - 1` переходов.

За маршрут он может получить банки из посещённых домов. Нужно минимизировать максимальный размер банки, который встретится на маршруте.


## 🎯 Задача

Нужно найти:

> минимальное возможное значение максимального размера банки на маршруте из `k` домов

Если невозможно сделать `k - 1` переходов, нужно вывести:

```text
-1
```


## 🔢 Формат входных данных

Первая строка:

```text
n m k
```

где:

* `n` — количество домов
* `m` — количество кабелей
* `k` — количество домов, которые Вова хочет посетить

Вторая строка:

```text
a1 a2 ... an
```

где `a[i]` — размер банки в доме `i`.

Далее идут `m` строк:

```text
u v
```

Это означает, что есть кабель:

```text
u → v
```


## 📤 Формат выходных данных

Вывести одно число:

* минимальный возможный максимальный размер банки
* или `-1`, если маршрут длиной `k - 1` переходов невозможен


## 💡 Идея задачи

Нужно минимизировать максимальное значение `a[i]` на маршруте.

Это удобно решать через:

> **бинарный поиск по ответу**


## 🧠 Ключевая идея

Пусть мы проверяем некоторое значение `limit`.

Оставим только те дома, где:

```text
a[i] <= limit
```

Теперь нужно понять:

> можно ли в этом подграфе пройти маршрут длиной хотя бы `k - 1` переходов?

Если можно, значит ответ не больше `limit`.

Если нельзя, значит `limit` слишком маленький.


## ⚙️ Проверка `can(limit)`

Строим подграф только из активных вершин:

```text
active[i] = true, если a[i] <= limit
```

Дальше есть два случая.


## 🔹 Случай 1. В активном подграфе есть цикл

Если в активном подграфе есть цикл, то по нему можно ходить сколько угодно раз.

Значит можно сделать маршрут любой большой длины.

👉 Тогда `can(limit) = true`.


## 🔹 Случай 2. Активный подграф — DAG

Если циклов нет, то нужно найти самый длинный путь в DAG.

Если длина самого длинного пути:

```text
>= k - 1
```

то маршрут существует.

Иначе — нет.


## ⚙️ Как проверить цикл и длину пути

Используем топологическую сортировку через алгоритм Кана:

1. Считаем входящие степени `indegree`
2. Кладём в очередь вершины с `indegree = 0`
3. Обрабатываем вершины
4. Одновременно считаем `longest[v]` — длину самого длинного пути до вершины `v`

Если после обработки остались необработанные активные вершины:

```text
processed < active_count
```

значит в графе есть цикл.


## 🔍 Бинарный поиск

Ответом может быть только одно из значений массива `a`.

Поэтому:

1. сортируем уникальные значения `a`
2. бинарным поиском ищем минимальное `limit`, для которого `can(limit) = true`


## 🧩 Почему это работает

Если маршрут возможен при некотором `limit`, то при большем `limit` он тоже возможен.

Потому что при увеличении `limit` активных домов становится только больше.

Значит условие монотонное, и можно использовать бинарный поиск.


## ⏱ Сложность

Пусть:

* `n` — количество домов
* `m` — количество кабелей

Проверка одного `limit`:

```text
O(n + m)
```

Бинарный поиск по значениям `a`:

```text
O(log n)
```

Итоговая сложность:

```text
O((n + m) log n)
```

Память:

```text
O(n + m)
```


## 📘 Пример

### Вход:

```text
5 6 3
1 10 4 6 3
1 2
2 3
3 4
4 5
5 3
2 4
```

### Выход:

```text
6
```


## 🚀 Важные моменты

* граф ориентированный
* маршрут может начинаться в любом доме
* нужно сделать именно `k - 1` переходов
* `k` может быть очень большим, до `10^18`
* если в подходящем подграфе есть цикл, можно набрать маршрут любой длины
* если цикла нет, считаем самый длинный путь в DAG


## 🎯 Зачем эта задача

* бинарный поиск по ответу
* работа с ориентированным графом
* поиск цикла через топологическую сортировку
* поиск самого длинного пути в DAG
* оптимизация под большие ограничения
