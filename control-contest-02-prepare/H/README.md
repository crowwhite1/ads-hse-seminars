# Задача H. Мисматч

## 📌 Условие

Есть граф:
- вершины — площадки (от `1` до `n`)
- рёбра — дороги с временем `w`

Два игрока:
- Мадин стартует из `1`
- Гоша стартует из `n`

Некоторые вершины — **мисматчи**:
- если игрок проходит через такую вершину  
→ он может **один раз** ускориться (все дальнейшие дороги делятся на 2)


## 🎯 Задача

Нужно:
> найти минимальное время, когда оба игрока смогут встретиться в одной вершине

Финальное время:
```

max(время Мадина, время Гоши)

```


## 🔢 Формат входных данных

- `t` — число тестов  
- для каждого теста:
  - `n, m, k`
  - список из `k` вершин (мисматчи)
  - `m` дорог `(u, v, w)`


## 📤 Формат вывода

Для каждого теста:
- минимальное время встречи
- или `-1`, если встречи нет


## 💡 Идея задачи

Это задача на:
> **кратчайшие пути с состоянием**


## 🧠 Ключевая идея

У каждой вершины есть **2 состояния**:
```

0 — без мисматча
1 — уже использовал мисматч

```

👉 значит, вместо обычного графа:
```

вершины × 2

```


## ⚙️ Алгоритм

Используем:
> **Дейкстру с состоянием**

Переход:
```

если мисматч ещё не использован:
можем использовать его в этой вершине

```

Вес:
```

если уже использовал мисматч:
w / 2
иначе:
w

```


## ⚙️ Решение

1. Запускаем Дейкстру:
   - от `1` → `dist_from_1`
   - от `n` → `dist_from_n`

2. Для каждой вершины `v`:
```

time_1 = min(dist_from_1[v][0], dist_from_1[v][1])
time_n = min(dist_from_n[v][0], dist_from_n[v][1])

```

3. Ответ:
```

min по всем v: max(time_1, time_n)

```id="answer"


## 🚫 Когда ответа нет

Если:
```

все значения = ∞

```id="no-answer"

👉 вывести `-1`


## ⏱ Сложность

- Дейкстра:
```

O((n + m) log n)

```

- на 2 запуска:
```

O((n + m) log n)

```


## 📘 Почему это работает

- мы учитываем, был ли использован мисматч
- Дейкстра на расширенном графе даёт корректные расстояния
- перебор вершины встречи даёт оптимум


## 🚀 Важные моменты

- мисматч можно использовать только **один раз**
- после использования ускорение действует **всегда**
- граф неориентированный
- веса чётные → деление безопасно


## 🎯 Навыки

- Дейкстра с состояниями
- моделирование состояний в графе
- минимакс-оптимизация (min max)
