# Задача D. Ну очень сложная задача

## 📌 Условие

Есть `n` компьютеров и `m` студентов.  
Каждый студент хранит свою работу на **двух компьютерах**.

У каждого компьютера есть время обновления `upd[i]` (от 0 до `h-1`), когда он **недоступен**.

Гарантируется:
- у каждого студента его два компьютера обновляются в **разное время**  
👉 значит всегда есть доступ хотя бы к одной копии


## 🎯 Задача

Нужно выбрать **минимальное множество компьютеров**, для которых:

👉 если сдвинуть время обновления на +1 (по модулю `h`)  
👉 **гарантия доступности не нарушится**


## 🔢 Формат входных данных

- `n, m, h`
- массив `upd[i]`
- `m` пар `(c1, c2)` — где лежат копии


## 📤 Формат выходных данных

- `k` — размер множества
- `k` компьютеров


## 💡 Идея задачи

Это задача на:
> **граф + сильно связанные компоненты (SCC)**


## 🧠 Ключевая идея

### 🔹 Строим граф

Для каждого студента `(u, v)`:

Добавляем ребро:
- `u → v`, если `(upd[u] + 1) % h == upd[v]`
- `v → u`, если `(upd[v] + 1) % h == upd[u]`

👉 это значит:
> если мы сдвинем `u`, он станет конфликтовать с `v`


## 🔥 Что означает ребро

Ребро `u → v`:
> если мы выберем `u`, то **нельзя выбирать `v`**


## ⚙️ Шаг 1. Найти SCC

Разбиваем граф на **сильно связанные компоненты**

👉 внутри SCC:
- все вершины взаимозависимы
- либо берем **всю компоненту**, либо **не берем**


## ⚙️ Шаг 2. Сжимаем граф

Строим граф компонент:

- вершины = SCC
- ребро между компонентами, если было ребро в исходном графе


## ⚙️ Шаг 3. Ищем "хорошие" компоненты

Нужны компоненты:

👉 **без исходящих рёбер**

Почему?
- если есть исходящее ребро → есть конфликт
- значит такую компоненту брать нельзя


## ⚙️ Шаг 4. Выбираем ответ

Среди компонент без исходящих рёбер:

👉 выбираем компоненту **минимального размера**


## 🧩 Почему это работает

- SCC объединяет взаимозависимые вершины
- граф компонент — DAG
- нам нужны "конечные" компоненты (sink components)

👉 они не зависят ни от кого


## ⏱ Сложность

- Время: **O(n + m)**
- Память: **O(n + m)**


## 📘 Пример

### Вход:
```

3 3 5
4 4 0
1 3
3 2
3 1

```

### Выход:
```

1
3

```


## 🚀 Важные моменты

- граф направленный
- используем **Косарайю / Тарьяна**
- обязательно учитывать `(upd[i] + 1) % h`
- ответ — **любая минимальная подходящая компонента**
