# Задача C. С Новым годом!

## 📌 Условие

Дан массив из `n` шариков, расположенных в линию.  
У каждого шарика есть число `a[i]`.

Цвет шарика определяется этим числом:
- **четное** → красный
- **нечетное** → синий


## 🎯 Задача

Для каждого шарика нужно найти:

👉 минимальное количество ходов, чтобы попасть на шарик **другого цвета**

Если это невозможно — вывести `-1`.


## 🔢 Правила переходов

Из позиции `i` можно перейти:
- в `i - a[i]`, если `i - a[i] ≥ 0`
- в `i + a[i]`, если `i + a[i] < n`


## 🔢 Формат входных данных

- `n (1 ≤ n ≤ 2 ⋅ 10^5)`
- массив `a` длины `n`, где `1 ≤ a[i] ≤ n`


## 📤 Формат выходных данных

Вывести массив `d`, где:
- `d[i]` — минимальное число ходов до шарика другого цвета
- или `-1`, если попасть нельзя


## 💡 Идея задачи

Наивный подход (BFS из каждой вершины) даст **O(n²)** → не пройдет.

👉 Нужно решить за **O(n)**.


## 🧠 Ключевая идея

### 🔹 Разворачиваем граф

Обычно:
- из `i` можно пойти в `j`

👉 мы строим **обратный граф**:
- из `j` можно прийти в `i`


### 🔹 Почему это важно?

Мы хотим:
> для каждой вершины найти ближайшую вершину другого цвета

👉 вместо `n` BFS:
делаем **2 BFS**:

1. от всех **четных** вершин
2. от всех **нечетных** вершин


## ⚙️ Алгоритм

### Шаг 1. Строим обратный граф

Для каждой вершины `i`:
- считаем `left = i - a[i]`
- считаем `right = i + a[i]`

Если переход возможен:
- добавляем `i` в список `reverse_graph[left/right]`


### Шаг 2. BFS по четности

Функция `bfs(parity)`:

- стартуем сразу из всех вершин с этой четностью
- расстояние до них = `0`
- обычный BFS по `reverse_graph`

👉 получаем расстояния до ближайшей вершины **этого типа**


### Шаг 3. Собираем ответ

- если вершина четная → берем расстояние до **нечетных**
- если нечетная → до **четных**


## 🧩 Почему это работает

Мы считаем:
- минимальное расстояние до всех вершин определенного типа

А затем просто выбираем нужный тип (противоположный)


## ⏱ Сложность

- Время: **O(n)**
- Память: **O(n)**


## 📘 Пример

### Вход:
```

5
5 4 2 5 2

```

### Выход:
```

-1 -1 1 -1 2

```


## 🚀 Важные моменты

- обязательно строить **обратный граф**
- BFS запускается сразу от множества вершин
- цвета определяются через `a[i] % 2`
- индексация — **0-based**


## 🎯 Зачем эта задача

- понимание BFS от множества источников
- работа с обратным графом
- оптимизация с O(n²) → O(n)
