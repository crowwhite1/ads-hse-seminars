# Задача I. Похорошела Москва при Собянине

## 📌 Условие

Дан город в виде прямоугольной сетки:

- `Z` — количество строк
- `X` — количество столбцов

Каждая клетка сетки — здание.

Между соседними зданиями могут быть уже проложены кабели:
- вниз
- вправо

Нужно добавить минимальное количество кабелей так, чтобы вся сеть стала связной.


## 🎯 Задача

Необходимо вывести:

- количество добавленных кабелей
- минимальную стоимость добавления
- список добавленных кабелей

Стоимость:
- вертикальный кабель — `1`
- горизонтальный кабель — `2`


## 🔢 Формат входных данных

Первая строка:

```text
Z X
````

Далее идут `Z` строк по `X` чисел.

Каждое число описывает уже существующие соединения из клетки `(q, w)`:

* `0` — нет кабеля ни вниз, ни вправо
* `1` — есть кабель вниз: `(q, w) → (q + 1, w)`
* `2` — есть кабель вправо: `(q, w) → (q, w + 1)`
* `3` — есть оба кабеля


## 📤 Формат выходных данных

Первая строка:

```text
A B
```

где:

* `A` — количество добавленных кабелей
* `B` — суммарная стоимость добавления

Далее `A` строк:

```text
q w e
```

где:

* `(q, w)` — координаты начального здания
* `e = 1` — добавить вертикальный кабель вниз
* `e = 2` — добавить горизонтальный кабель вправо


## 💡 Идея задачи

Это задача на построение минимального набора соединений между компонентами.

Так как уже существующие кабели бесплатные, сначала нужно объединить здания, которые уже соединены.

После этого нужно добавить недостающие кабели так, чтобы соединить все компоненты с минимальной стоимостью.

👉 Это похоже на построение **минимального остовного дерева**.


## 🧠 Ключевая идея

Используем:

> **DSU + алгоритм Крускала**

DSU помогает хранить компоненты связности.


## ⚙️ Алгоритм

### Шаг 1. Пронумеровать здания

Каждую клетку `(row, column)` переводим в один номер:

```text
id = row * X + column
```

Так удобнее работать с DSU.


### Шаг 2. Объединить уже соединённые здания

Для каждой клетки смотрим значение:

* если есть кабель вниз (`value & 1`) → объединяем текущую клетку с нижней
* если есть кабель вправо (`value & 2`) → объединяем текущую клетку с правой


### Шаг 3. Собрать возможные новые кабели

Для каждой клетки:

* если вниз кабеля нет, но нижняя клетка существует → можно добавить кабель стоимостью `1`
* если вправо кабеля нет, но правая клетка существует → можно добавить кабель стоимостью `2`


### Шаг 4. Отсортировать кабели по стоимости

Сначала выгоднее добавлять вертикальные кабели, потому что они стоят `1`.

Горизонтальные стоят `2`.


### Шаг 5. Крускал

Проходим по всем возможным кабелям:

* если кабель соединяет разные компоненты → добавляем его в ответ
* если здания уже в одной компоненте → кабель не нужен


## 🧩 Почему это работает

Нужно соединить все здания с минимальной стоимостью.

Уже существующие кабели считаются бесплатными, поэтому они сразу объединяют компоненты.

Дальше задача сводится к выбору самых дешёвых кабелей, которые соединяют разные компоненты.

Это ровно логика алгоритма Крускала.


## ⏱ Сложность

Пусть всего зданий `Z * X`.

* Время: **O(Z * X log(Z * X))**
* Память: **O(Z * X)**

Сортировка нужна для возможных новых кабелей.

### Выход:

```text

```


## 🚀 Важные моменты

* Координаты в ответе выводятся с **1**
* В коде удобнее использовать индексацию с **0**
* `e = 1` — кабель вниз
* `e = 2` — кабель вправо
* Если сеть уже связная, нужно вывести `0 0`
