# Задача B. Мосты и острова

## 📌 Условие

В королевстве есть `N` островов. Между некоторыми островами уже построены мосты, а между некоторыми — нет.

Известно:
- стоимость строительства моста между любой парой островов
- какие мосты уже существуют

Необходимо достроить недостающие мосты так, чтобы:
- все острова оказались связаны (можно через другие острова)
- суммарная стоимость была **минимальной**


## 🔢 Формат входных данных

1. Число `N` `(1 ≤ N ≤ 50)` — количество островов  
2. Матрица `N × N` — стоимости мостов  
3. Матрица `N × N` — наличие мостов:
   - `1` — мост уже есть  
   - `0` — моста нет  

Обе матрицы симметричны.


## 📤 Формат выходных данных

Вывести одно число — минимальную стоимость, необходимую для соединения всех островов.


## 💡 Идея задачи

Это классическая задача на **минимальное остовное дерево (MST)**.

Но есть важное отличие:
- часть рёбер уже **построена (их стоимость = 0)**


## 🧠 Как мыслить

### 🔹 Шаг 1. Что уже есть

Если мост уже существует → эти вершины уже соединены  
👉 объединяем их в одну компоненту связности


### 🔹 Шаг 2. Что нужно сделать

Осталось соединить все компоненты между собой с минимальной стоимостью.

👉 это классическая задача MST


## ⚙️ Алгоритм (Крускал + DSU)

1. Создаём структуру **DSU (Disjoint Set Union)**
2. Обрабатываем существующие мосты:
   - если `1` → объединяем вершины
3. Формируем список всех **недостающих рёбер**:
   - добавляем `(стоимость, u, v)`
4. Сортируем рёбра по стоимости
5. Проходим по ним:
   - если вершины в разных компонентах → объединяем и добавляем стоимость


## 🧩 Почему это работает

Мы:
- сначала учитываем уже построенные мосты (бесплатно)
- затем достраиваем минимально возможный набор рёбер

👉 это и есть построение MST с уже существующими компонентами


## ⏱ Сложность

- Время:  
  **O(N² log N)** — из-за сортировки рёбер  
- Память:  
  **O(N²)** — хранение матрицы и рёбер  


## 📘 Пример

### Вход:
```

3
0 1 20
1 0 10
20 10 0
0 1 0
1 0 0
0 0 0

```

### Выход:
```

10

```


## 🚀 Важные моменты

- Граф неориентированный (матрицы симметричны)
- Не нужно учитывать рёбра дважды → берём только `j > i`
- Уже существующие мосты **не добавляют стоимость**
- Нужно соединить **все компоненты**


## 🎯 Зачем эта задача

- Практика алгоритма Крускала
- Работа с DSU (union-find)
- Понимание минимального остовного дерева
